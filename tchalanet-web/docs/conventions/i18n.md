# i18n Convention

> Status: DRAFT v0.1  
> Scope: local translations, backend overrides, language switching

## Rule

i18n is resolved by merging frontend local bundles with backend overrides. Local bundles provide safe fallback; backend overrides win when the same key exists.

## Placement

```text
apps/tch-portal/src/app/core/i18n/                   loader, merge, language state
apps/tch-portal/public/assets/i18n/                  local fallback bundles
libs/shared-config/src/lib/runtime/runtime-paths.ts stable i18n configuration
```

Use `core/i18n` for:

- language state;
- translate loader;
- backend override loading;
- merge rules;
- language switcher integration.

Use assets for local fallback bundles.

`PORTAL_I18N_CONFIG` in `@tch/shared-config` owns:

- default and fallback language;
- local bundle prefix/suffix;
- backend path;
- ordered surface list.

`app.config.ts` wires this configuration into `MergedTranslateLoader`; it does not duplicate values.

## Runtime Paths

```text
local fallback: /assets/i18n/{lang}.json
backend merge:  GET /api/v1/public/i18n?locale={lang}&surface=PUBLIC_COMMON&surface=PUBLIC_HOME
```

The backend path is `API_PATHS.i18n.public`. Browser calls remain relative so `nx serve` can proxy
them to the configured local backend.

## Backend Response

The backend i18n response may group keys by surface:

```json
{
  "locale": "fr",
  "surfaces": {
    "PUBLIC_HOME": {
      "home.hero.title": "Bienvenue"
    }
  }
}
```

The frontend flattens all surface maps and merges the keys into the active language bundle.

## Merge Order

```text
local fallback bundle
then backend overrides
```

Backend wins on collision.

Missing backend response must not break rendering when local translations exist.

## Keys

Use stable dot keys:

```text
home.hero.title
home.hero.subtitle
error.generic
theme.presets.tchalanet
```

PageModel and API payloads reference i18n keys. They do not embed translated display text unless the backend contract explicitly says the text is already localized.

## Language State

The selected language is runtime state. Components use translation pipes/services; they do not fetch translation JSON directly.

## Anti-Patterns

Do not:

- duplicate the same key under many surface prefixes without need;
- put tenant copy in frontend source when it should be backend override;
- let PageModel carry translation maps;
- block the whole app when override loading fails;
- concatenate translated fragments when one full sentence key is possible.
