## Acceptance checklist

- [ ] `features.cashier` has zero imports from `core.sales.internal.*`.
- [ ] Preview and sell both call `SaleAcceptanceEvaluator`.
- [ ] Preview is read-only and returns best-effort warning.
- [ ] Sell uses idempotency and rechecks exposure in final mode.
- [ ] `PENDING_APPROVAL` is not exposed to cashier POS.
- [ ] `APPROVAL_REQUIRED` appears as `REQUIRES_CHANGES` in POS preview/sell responses.
- [ ] Accepted sell response includes populated `backup`.
- [ ] `backup.shareableText` equals system-send message body byte-for-byte.
- [ ] `EXPOSURE_CHANGED` appears when preview is invalidated by concurrent sales.
- [ ] Cancel window defaults to 3 minutes and releases exposure.
- [ ] Send dedup prevents duplicate sends within 60 seconds.
- [ ] SMS/WhatsApp send text only.
- [ ] Money serializes as `X.XX CURRENCY`.
- [ ] Public verification normalizes dashed and non-dashed codes.
- [ ] Print returns file responses with no API wrapper.
- [ ] Required E2E tests pass.
