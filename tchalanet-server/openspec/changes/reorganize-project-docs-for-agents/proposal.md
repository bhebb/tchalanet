# Proposal — Reorganize Documentation for AI Agents and Developers

## Problem

After the platform/modulith refactor, documentation must be easier to locate, update, and trust. AI agents need short operational rules close to code and stable architecture docs that do not drift.

## Goals

- Separate stable architecture decisions from living module inventories.
- Provide agent-oriented README files per module/capability.
- Make docs easy to update after code changes.
- Provide clear doc ownership and update triggers.

## Non-goals

- Do not generate massive prose nobody reads.
- Do not duplicate full ADR content in every module README.
