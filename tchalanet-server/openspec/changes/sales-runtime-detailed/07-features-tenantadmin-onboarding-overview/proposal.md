# Change: features.tenantadmin Onboarding + Overview

## Why

Tenant admin needs a guided setup and overview, but the feature must not replace core CRUD endpoints.

## What

Create/align `features.tenantadmin` for:

- onboarding checklist
- setup progress
- config overview
- operational overview
- shortcuts/actions metadata

## Non-goals

- Do not implement outlet/terminal/policy CRUD here.
- Do not make feature layer source of truth.
