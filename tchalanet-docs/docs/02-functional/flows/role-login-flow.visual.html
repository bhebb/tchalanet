📌 Règles à ne pas casser

  • Request Context est universel. Operational Context est POS/terrain, pas global.
  • Operational Context est attaché tôt mais validé tard, par action.
  • Le controller reçoit @CurrentContext. Les handlers reçoivent les champs via commands/queries.
  • Les batchs construisent explicitement un contexte avec BatchTchContextBinder.
  • Le tenant ne vient jamais du body client comme vérité.
  • Les events/async ne doivent pas dépendre d'un ThreadLocal HTTP invisible.
  • Un flow compose des sous-flows par liens. Il ne les recopie pas.
  • Ce HTML est une vue pédagogique, pas une source de vérité concurrente.
